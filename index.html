// API Reference

TopUp & Voucher
API v4.0.0

Unified endpoint for Free Fire shell top-up and UniPin voucher redemption. Supports BD, SG, and INDO regions. All orders process asynchronously — results delivered via HTTP callback.

Base URL https://your-server.com
// Authentication
API Token

Every request must include a valid API token. Three methods are supported — the first found is used.

MethodWhereExample
Authorization Header HTTP Header Authorization: Bearer your_token
Query Parameter URL Query ?token=your_token
Body Field JSON Body "api_key": "your_token"

// Main Endpoint
POST /topup

Single endpoint for all order types. Returns immediately with an accepted status. The actual top-up runs in the background — result is posted to your callback URL.

url is required. Without a callback URL you will not receive the order result. Always include your HTTPS endpoint in the url field.

Request Fields

FieldTypeDescription
orderid required string Your unique order identifier
playerid required string Free Fire UID (min 8 digits for voucher orders)
code required string Order type key: shell / sgshell / indo, or voucher serial(s)
url required string Your HTTPS callback URL to receive the order result
api_key required* string API token — or pass via Authorization header / query param
pacakge shell only string Package to purchase. Required for all shell orders. Spelling: pacakge (intentional — keep it as-is)
qty optional int 1–5 How many of the same shell package to buy in one order. Default 1, max 5. For vouchers, qty is set by the number of codes in the code field.
username shell only string Garena account username
password shell only string Garena account password
autocode shell only string TOTP 2FA secret key
metadata optional object Custom key-value pairs — forwarded unchanged in the callback

Immediate Response

{
  "status":  "accepted",   // "accepted" for shell, "processing" for voucher
  "orderid": "ORD-001",
  "message": "Processing Weekly Lite x1 on BD shop."
}
// Order Type
Voucher Redemption

Redeem UniPin vouchers by Free Fire UID. Send one or up to 5 vouchers comma-separated in the code field.

Code format: SERIAL:PIN or SERIAL PIN. PIN must be exactly 16 digits (dashes are stripped automatically). For multiple vouchers, separate with commas. Max 5 per request.

Single Voucher

{
  "orderid":  "ORD-001",
  "playerid": "123456789",
  "code":     "BDMB-T-S-01488799:6413243937299114",
  "api_key":  "your_token",
  "url":      "https://your-site.com/callback"
}

Multiple Vouchers (up to 5)

{
  "orderid":  "ORD-002",
  "playerid": "123456789",
  "code":     "BDMB-T-S-01488799:6413243937299114, UPBD-Q-S-01565164:6443465646246675",
  "api_key":  "your_token",
  "url":      "https://your-site.com/callback"
}

// Callbacks
Callback Formats

When an order completes, the API POSTs JSON to your url. Your server must respond with HTTP 200.

✅
success
All items completed successfully
❌
failed
All items failed
⚠️
partial
Mixed — some ok, some failed

Voucher — Single Success

{
  "status":   "success",
  "orderid":  "ORD-001",
  "nickname": "PlayerName",
  "content":  "✅ BDMB-T-S-01488799 6413-2439-3729-9114 (Weekly Lite)"
}

Voucher — Single Failed

{
  "status":   "failed",
  "orderid":  "ORD-001",
  "nickname": "N/A",
  "content":  "❌ BDMB-T-S-01488799 — Voucher already used"
}

Voucher — Multi / Partial

{
  "status":   "partial",
  "orderid":  "ORD-002",
  "nickname": "PlayerName",
  "content": {
    "batch": [
      { "uc": "BDMB-T-S-01488799 6413-2439-3729-9114", "ok": true,  "detail": "✅ Success" },
      { "uc": "UPBD-Q-S-01565164 6443-4656-4624-6675", "ok": false, "detail": "❌ Invalid voucher" }
    ]
  }
}

Shell — Success

{
  "status":   "success",
  "orderid":  "ORD-003",
  "nickname": "PlayerName",
  "content": {
    "batch": [
      { "package": "LITE", "ok": true, "detail": "Purchase successful" }
    ]
  }
}

Shell — Failed

{
  "status":   "failed",
  "orderid":  "ORD-003",
  "nickname": "N/A",
  "content": {
    "batch": [
      { "package": "LITE", "ok": false, "detail": "Insufficient balance" }
    ]
  }
}

With Metadata (any type)

Metadata pass-through: Any metadata object sent in the request is forwarded unchanged in the callback payload.
{
  "status":   "success",
  "orderid":  "ORD-010",
  "nickname": "PlayerName",
  "content":  "✅ BDMB-T-S-01488799 6413-2439-3729-9114 (Weekly Lite)",
  "metadata": {
    "user_id": "usr_99",
    "ref":     "TXN-XYZ"
  }
}
