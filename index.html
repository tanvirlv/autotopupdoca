Quick Start

1. Get Your API Token

Contact the administrator to receive your API token with custom usage limits.

2. Make Your First Request

curl -X POST http://161.248.189.71:8003/topup \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "orderid": "ORDER_12345",
    "playerid": "1234567890",
    "code": "BDMB-T-S-XXXXX,BDMB-U-S-YYYYY",
    "url": "https://your-callback.com/webhook"
  }'

3. Response

{
  "status": "accepted",
  "orderid": "ORDER_12345",
  "message": "Order accepted. 2 code(s) will be redeemed."
}

API Endpoints

POST /topup

Process UC code redemption order

Authentication

Include your API token in one of these ways:

  • Header: Authorization: Bearer YOUR_API_TOKEN
  • Query: ?token=YOUR_API_TOKEN

Request Body

Parameter Type Required Description
orderid string Required Unique order identifier
playerid string Required Player/User ID
code string Required Comma-separated UC codes (max 5)
url string Required Callback URL for status updates
metadata object Optional Custom data for tracking

Supported Code Formats

BDMB Series: BDMB-T-S-, BDMB-U-S-, BDMB-J-S-, BDMB-I-S-, BDMB-Q-S-, BDMB-K-S-, BDMB-S-S-, BDMB-L-S-, BDMB-M-S-

UPBD Series: UPBD-Q-S-, UPBD-R-S-, UPBD-G-S-, UPBD-F-S-, UPBD-N-S-, UPBD-H-S-, UPBD-P-S-, UPBD-I-S-, UPBD-J-S-, UPBD-7-S-

Responses

Status Description
200 OK Order accepted successfully
400 Bad Request Invalid parameters or code format
401 Unauthorized Invalid or missing API token
429 Too Many Requests Usage limit exceeded

PHP

]);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode([

    'code' => 'BDMB-T-S-XXXXX,BDMB-U-S-YYYYY',

]));

$response = curl_exec($ch);
curl_close($ch);

print_r(json_decode($response));

Best Practices

🔐 Security

  • Store tokens in environment variables
  • Never expose tokens in client-side code
  • Use HTTPS for all API calls

⚡ Performance

  • Batch codes when possible (up to 5)
  • Implement retry logic with backoff
  • Log all API calls for debugging

📊 Monitoring

  • Track your token usage regularly
  • Set up alerts for limit thresholds
  • Monitor webhook delivery rates
